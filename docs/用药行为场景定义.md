# 小猫吃药监督 — 用药行为场景定义

> 本文档定义用户在「用药管理」中的所有日常行为场景，作为功能完善的依据。
> 每个场景标注当前实现状态：✅ 已实现 / ⚠️ 部分实现 / ❌ 未实现

---

## 一、药品管理（CRUD）

### 1.1 添加药品 ✅
- **触发**：药柜页点「+ 添加」→ 进入添加表单
- **可填字段**：
  - 药品名称（必填）
  - 剂量（如 200mg）
  - 单位（粒/片/颗/ml/滴/包/支）
  - 服用频率（每天一次/两次/三次/每周/按需/周期服用）
  - 提醒时间（根据频率自动生成默认时间，可修改）
  - 标签颜色（8色可选）
  - 有效期（日期选择器）
  - 备注（如「饭后服用」）
  - 药品照片（拍照或从相册选）
- **保存后**：自动创建对应的 schedules 记录

### 1.2 查看药品详情 ✅
- **触发**：药柜列表点某个药品 → 进入详情页
- **展示内容**：药名、剂量、频率、提醒时间、有效期倒计时、备注、照片

### 1.3 编辑药品 ❌
- **触发**：详情页应有「编辑」按钮 → 进入编辑表单
- **可改字段**：同添加，所有字段都可修改
- **修改后**：更新 medications 表 + updated_at
- **当前问题**：详情页只有「停用」按钮，无编辑入口
- **需要新增**：
  - `app/medication/edit.tsx` 编辑页面（复用 add.tsx 的表单逻辑）
  - 详情页增加「编辑」按钮
  - 编辑时同步更新 schedules（如果频率或时间变了）

### 1.4 停用药品 ✅
- **触发**：详情页「停用此药品」/ 药柜列表长按
- **行为**：软删除（is_active = false），不再出现在今天列表和药柜中
- **确认**：弹窗二次确认

### 1.5 恢复已停用药品 ❌
- **场景**：用户停用了某个药，后来又需要吃了
- **需要新增**：
  - 药柜页增加「已停用」筛选/入口
  - 已停用药品列表，可点击「重新启用」
  - 恢复 = set is_active = true

### 1.6 永久删除药品 ❌
- **场景**：用户不再需要这个药品记录（区别于停用）
- **行为**：硬删除 medication + 关联的 schedules + logs
- **注意**：需弹窗强提示「删除后不可恢复，包括所有历史记录」
- **入口**：已停用药品列表中提供「永久删除」按钮

## 二、用药计划 / 时间表管理

### 2.1 设置提醒时间 ⚠️
- **当前**：添加药品时自动根据频率生成默认时间
- **缺失**：
  - 添加完药品后无法单独修改某个提醒时间
  - 无法增加/删除单个提醒时间点
- **需要新增**：
  - 详情页/编辑页可增删改 schedules
  - 每个 schedule 可单独启用/禁用
  - 修改 schedule 时同步更新本地通知

### 2.2 设置提醒日期（星期几）⚠️
- **当前**：默认每天（[0,1,2,3,4,5,6]），weekly 模式也是全天
- **缺失**：
  - 无 UI 让用户选择「只在工作日」或指定周几
  - weekly 模式应让用户选某一天（如每周一）
- **需要新增**：
  - 添加/编辑时提供星期几选择器（周一到周日 chip 多选）
  - weekly 模式下默认单选一个

### 2.3 暂停/恢复某个提醒 ⚠️
- **当前**：schedule 表有 enabled 字段，但前端无操作入口
- **需要新增**：
  - 详情页每个提醒旁边加个开关（Switch）
  - 关闭 = 暂时不提醒但保留配置
  - 关闭时取消对应本地通知

### 2.4 周期服用（时间段 + 循环）❌
- **场景**：某些药需要在一段时间内服用，然后停药，然后重复。典型例子：
  - 激素药：每月第 1~21 天吃，停 7 天，下月重复
  - 抗生素疗程：连续吃 7 天然后停
  - 化疗周期：吃 14 天停 7 天，循环
- **数据模型新增字段**（medications 表）：
  - `cycle_type`: 'none' | 'once' | 'monthly' | 'custom'
    - `none`：普通模式（现有逻辑）
    - `once`：单次疗程（从 A 日到 B 日，结束后自动停用）
    - `monthly`：每月重复（每月第 X 天到第 Y 天）
    - `custom`：自定义循环（吃 N 天，停 M 天，循环）
  - `cycle_start_day`: integer — 月循环的开始日（如 1 = 每月 1 号）
  - `cycle_end_day`: integer — 月循环的结束日（如 21 = 每月 21 号）
  - `cycle_active_days`: integer — 自定义循环中「吃药天数」（如 14）
  - `cycle_rest_days`: integer — 自定义循环中「停药天数」（如 7）
  - `cycle_start_date`: date — 疗程/循环的起始日期
  - `cycle_end_date`: date | null — 单次疗程的结束日期（循环模式为 null = 无限循环）
- **前端 UI**：
  - 添加/编辑药品时，频率选「周期服用」→ 展开周期设置区域
  - 周期类型选择：单次疗程 / 每月重复 / 自定义循环
  - 每月重复：两个日期选择器「每月第 __ 天 到 第 __ 天」
  - 自定义循环：两个数字输入「吃 __ 天，停 __ 天」
  - 起始日期选择器（默认今天）
  - 单次疗程额外显示结束日期
- **核心逻辑变更**：
  - `fetchTodayItems` 需要判断今天是否在周期的「活跃期」内
    - 月循环：今天的日期号 >= cycle_start_day && <= cycle_end_day
    - 自定义循环：从 cycle_start_date 开始算，(天数差 % (active + rest)) < active 则为活跃
    - 不在活跃期 → 跳过该药品，不生成 log
  - 药柜列表显示当前周期状态：「服用中（第 5/21 天）」或「停药中（还有 3 天恢复）」
  - 今天页面：周期药品额外显示进度信息「第 X/N 天」
- **示例**：
  - 用户添加「优思明」，频率选「周期服用」→ 类型选「每月重复」
  - 设置：每月第 1 天到第 21 天，每天 1 粒，晚上 21:00
  - 效果：每月 1~21 号今天页出现此药，22~31 号不出现
  - 药柜页显示：「优思明 · 每月第1-21天 · 服用中（第 15/21 天）」

### 2.5 临时加服（按需服用）❌
- **场景**：用户临时需要吃一次药，不在固定计划内（如止痛药）
- **需要新增**：
  - 「按需服用」频率的药品，今天页可手动「记一笔」
  - 或者在今天页增加「+ 临时服药」按钮
  - 创建一条 schedule_id = null 的 log 记录

## 三、每日用药操作

### 3.1 查看今日用药列表 ✅
- **触发**：打开 App → 今天 Tab
- **展示**：按时间段分组（早上/下午/晚上），每个药品卡片显示名称、剂量、时间
- **状态显示**：pending（待服）/ taken（已吃）/ skipped（跳过）

### 3.2 标记「已吃」 ✅
- **触发**：点药品卡片的「吃了」按钮
- **行为**：log.status = 'taken'，记录 taken_at 时间戳
- **反馈**：触觉反馈 + 卡片变灰 + 打勾

### 3.3 标记「跳过」 ✅
- **触发**：点药品卡片的「跳过」按钮
- **行为**：log.status = 'skipped'

### 3.4 撤回操作 ✅
- **触发**：点已完成/已跳过的药品 → 恢复为 pending
- **行为**：log.status = 'pending'，taken_at = null

### 3.5 标记「漏服」 ❌
- **场景**：当天过了计划时间但没有操作 → 应自动标记为漏服
- **当前**：过期的 pending log 永远停留在 pending 状态
- **需要新增**：
  - 后台定时检查：如果 scheduled_time 过去超过 2 小时且仍 pending → 标记为 missed
  - 或在 fetchTodayItems 时判断：当前时间 > scheduled_time + 2h 且 pending → 前端显示为「漏服」
  - 漏服的卡片显示不同样式（红色/灰色）
  - 漏服后仍可手动改为「补吃了」

### 3.6 补服记录 ❌
- **场景**：用户漏服后补吃了
- **需要新增**：
  - 漏服卡片上提供「补吃了」按钮
  - 记录 taken_at（实际补服时间），status = 'taken'
  - 统计时区分「按时服用」和「补服」（可通过 taken_at vs scheduled_time 对比判断）

### 3.7 全天完成庆祝 ✅
- **当前**：全部 taken 后显示 🎉「今天的药都吃完啦！」

## 四、用药记录与统计

### 4.1 查看某天的用药记录 ✅
- **触发**：记录 Tab → 14 天日历条 → 点某天
- **展示**：当天每条 log 的药名、计划时间、实际时间、状态图标

### 4.2 查看更长时间范围 ❌
- **当前**：只能看最近 14 天
- **需要新增**：
  - 日历条可左右滑动查看更早日期
  - 或改为月历视图，每天格子上标记完成率颜色
  - 可按月份跳转

### 4.3 连续打卡天数 ✅
- **当前**：streak 逻辑已实现，在今天页和记录页显示

### 4.4 用药完成率统计 ⚠️
- **当前**：记录页只显示某一天的完成率
- **需要新增**：
  - 周报：过去 7 天的整体完成率
  - 月报：过去 30 天的整体完成率
  - 单药统计：某个药的历史完成率（连续服用了多少天、漏服了几次）
  - 可视化：柱状图或折线图展示趋势

### 4.5 导出用药记录 ❌
- **场景**：用户看医生时展示用药记录
- **需要新增**：
  - 按日期范围导出（PDF / 分享图片 / 纯文本）
  - 内容包括：药品名、计划时间、实际服用时间、状态
  - 可分享给医生/家人

## 五、信息修改与删除

### 5.1 修改用药记录 ❌
- **场景**：用户误操作标记了「已吃」想改成「跳过」，或修改实际服药时间
- **当前**：只能撤回（undo → pending），然后重新操作
- **需要新增**：
  - 记录页长按某条 log → 弹出选项：改为已吃 / 改为跳过 / 改为漏服 / 修改服药时间

### 5.2 删除某天的用药记录 ❌
- **场景**：某天的 log 数据有误，整天删除
- **需要新增**：
  - 记录页某天 → 操作菜单「清除当天记录」
  - 删除后，如果有有效 schedule，下次访问时会重新生成 pending log

### 5.3 批量管理药品 ❌
- **场景**：用户药品多了，想批量停用/删除
- **需要新增**：
  - 药柜页进入「编辑模式」→ 多选 → 批量停用/删除

### 5.4 修改账号信息 ❌
- **当前**：设置页只显示邮箱 + 退出
- **需要新增**：
  - 修改密码
  - （可选）修改昵称/头像

## 六、提醒与通知

### 6.1 本地推送通知 ⚠️
- **当前**：notification 模块代码已写，但未在添加药品时实际调用 scheduleMedicationReminder
- **需要新增**：
  - 添加/编辑药品时，自动注册本地通知
  - 删除/停用药品时，自动取消对应通知
  - 修改提醒时间时，取消旧通知 + 注册新通知

### 6.2 通知点击跳转 ❌
- **场景**：用户点推送通知 → 应跳到今天页，高亮对应药品
- **需要新增**：
  - 在 _layout.tsx 监听 notification response
  - 根据 notification data 跳转到对应页面

### 6.3 提前提醒 ❌
- **场景**：用户想提前 5/10/15 分钟收到提醒
- **需要新增**：
  - 设置页增加「提前提醒」选项（0/5/10/15/30 分钟）
  - 调整 notification trigger 的时间
