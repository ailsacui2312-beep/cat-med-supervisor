# 小猫吃药监督 — UI 设计描述（给 Google Stitch 用）

## 产品概述

一款温暖风格的用药管理 App，帮助用户追踪每日用药、记录身体数据（血糖/血压/体重）。设计风格偏健康+可爱，主色调为暖橙色，整体轻松不冷冰冰。

**目标用户**：需要长期服药的年轻人（如激素类周期用药、慢性病管理）
**平台**：iOS / Android（React Native）

---

## 色彩系统

- **主色**：#FF9F43（暖橙色）
- **背景**：#FFF8F0（奶油白）、卡片用纯白 #FFFFFF
- **成功/已服药**：#4CAF50（绿）
- **危险/漏服**：#FF6B6B（红）
- **警告/跳过**：#FFD93D（黄）
- **文字**：#2D2D2D（主要）、#666（次要）、#999（辅助）
- **边框**：#E8E0D8（浅色分隔线）

风格关键词：温暖、圆润、亲和、健康感、不冷冰冰

## 页面 1：今天（首页 / Today Tab）

这是用户每天打开 App 看到的第一个页面。

**顶部区域**：
- 左侧显示今天日期（如 "3月16日 周日"）
- 右侧显示连续服药天数徽章（🔥 火焰 emoji + "连续 7 天"）
- 下方一条进度条：已服 / 总数（如 "2/5"），绿色填充

**主体内容 — 按时间段分组的药品卡片**：
- 分为三组：🌅 早上、☀️ 下午、🌙 晚上
- 每张卡片内容：
  - 左侧：彩色圆点（用户自选的药品颜色）
  - 中间：药名、剂量（如 "1粒"）、计划时间（如 "08:00"）
  - 右侧：操作按钮
    - 待服药状态：橙色 "吃了" 按钮 + 灰色 "跳过" 按钮
    - 已服状态：绿色 ✓ 按钮，文字加删除线，卡片半透明
    - 漏服状态：红色背景色调，⚠️ 标签 + 黄色 "补吃了" 按钮
    - 已跳过状态：灰色 "已跳过" 按钮

**全部完成状态**：
- 🎉 大 emoji + "今天的药都吃完啦！" 庆祝文案

**空状态**：
- 🐱 猫猫 emoji + "还没有添加药品哦" + 橙色按钮 "去添加"

---

## 页面 2：药柜（Medicine Cabinet Tab）

管理所有药品 + 查看身体数据的页面。

**顶部**：
- 标题 "药柜"
- 右上角 "+ 添加" 按钮（橙色文字）

**药品列表**：
- 每张卡片：
  - 左侧彩色竖条（5px 宽，药品颜色）
  - 药品名称（17px 粗体）
  - 副标题行：剂量 + 单位 + 频率文字（如 "1粒 · 每日一次"）
  - 如果是周期用药：🔄 橙色状态文字（如 "每月1-21日 · 第 15/21 天"）
  - 提醒时间行：⏰ + 时间列表
  - 如果快过期：⚠️ 黄色/红色过期提醒标签
  - 右侧 › 箭头
- 长按卡片 → 弹出停用确认

**身体数据区域**（在药品列表下方）：
- "身体数据" 标题 + 右侧 "+ 记录" 链接
- 三张小卡片并排：
  - 💉 血糖：最新值 + 颜色表示正常/偏高/偏低 + 时间
  - 🫀 血压：如 "120/80 mmHg" + 状态颜色 + 时间
  - ⚖️ 体重：如 "65.5 kg" + 时间
  - 无数据时显示 "暂无数据" 灰色文字
- 下方一个按钮："📷 截图识别血糖"

**已停用药品区域**（折叠）：
- 点击 "已停用 (2) ▼" 展开
- 灰色半透明卡片 + "恢复" / "删除" 按钮

## 页面 3：记录（Stats Tab）

用药历史和统计页面。

**内容**：
- 📊 用药统计图表
- 按日/周/月的服药率
- 历史记录列表

---

## 页面 4：添加药品（Add Medication）

模态页面，从药柜点 "+ 添加" 进入。

**顶部导航**：取消 | "添加药品" | 保存

**表单内容（从上到下）**：

1. **药品拍照**：虚线方框 100x100，📸 + "拍照记录药品"
2. **药品名称**：文本输入框
3. **剂量 + 单位**：数字输入 + 横向可滑动标签（粒、片、次、ml、mg、滴、贴、喷）
4. **服药频率**：横向标签选择
   - 每日一次、每日两次、每日三次、每周、按需、周期服用
   - 选中标签：橙色填充背景
5. **周期设置**（仅在选择 "周期服用" 时显示）：
   - 三种模式：单次疗程 / 每月重复 / 自定义循环
   - 每月重复：输入第几天到第几天（如 1-21 日）
   - 自定义：X 天吃药 + Y 天休息
6. **提醒时间**：显示时间按钮，可添加/删除/修改
7. **颜色选择**：8 个彩色圆点（32px），选中有边框
8. **有效期**：日期选择器，可选填
9. **备注**：多行文本框

---

## 页面 5：记录身体数据（Add Health Data）

从药柜页身体数据区进入。

**顶部**：取消 | "记录身体数据" | 保存

**类型选择**：三张卡片并排
- 💉 血糖 / 🫀 血压 / ⚖️ 体重
- 选中状态：对应颜色的边框 + 淡色背景

**数值输入**：
- 大号输入框（24px 粗体字），右侧显示单位
- 血压时有两个输入框（收缩压 + 舒张压）

**实时状态指示**：
- 输入后立即显示 "正常" / "偏高" / "偏低" 彩色标签

**备注**：多行输入，如 "餐前、空腹、运动后..."

---

## 页面 6：截图识别血糖（Screenshot Recognition）

用 AI 识别血糖仪 App 截图。

**步骤一 — 选择图片**：
- 居中大 📸 emoji
- "选择血糖仪截图" 标题
- "支持欧态（Ottai）等 CGM App 的截图" 副标题
- 两个按钮："从相册选择"（实心橙色）+ "拍照"（描边橙色）

**步骤二 — 预览 + 识别**：
- 图片预览（圆角，全宽，高 280px）
- "换一张" 文字链接
- "开始识别" 大按钮
- 识别中：转圈动画 + "AI 识别中..."

**步骤三 — 结果确认**：
- 白色结果卡片：
  - "识别结果" 标题
  - 💉 血糖值：大字显示数值
  - 状态：颜色文字
  - 其他信息：TIR、趋势等
  - 置信度：高/中/低
- "确认保存" 按钮

---

## 页面 7：身体数据历史（Health History）

点击药柜页的健康卡片进入，按类型展示。

**顶部**：← 返回 | "💉 血糖记录" | "+ 记录"

**记录卡片列表**：
- 左侧：数值（18px 粗体）+ 状态标签（正常/偏高/偏低）
- 右侧：日期（MM月dd日）+ 时间（HH:mm）
- 如果来源是截图识别：显示 "截图识别" 小标签
- 如有备注：分隔线下方显示备注

**交互**：长按删除、下拉刷新

**空状态**：大 emoji + "暂无XX记录" + 提示文字

---

## 底部 Tab 栏

- 高度 72px，白色背景，顶部浅色阴影
- 三个 Tab：
  - 💊 今天
  - 🗄️ 药柜
  - 📊 记录
- 未选中：50% 透明度，灰色文字
- 选中：100% 透明度，橙色文字，粗体

---

## 设计风格关键词总结

- 圆角卡片（14px radius）
- 大量 emoji 作为视觉元素（不用 icon 库）
- 暖色调（橙色系 + 奶油白背景）
- 状态颜色直觉化（绿=好、红=注意、黄=警告）
- 表单元素大且易触摸（48px+ 高度）
- 整体感觉：健康 App 但不冷冰冰，像一个关心你的朋友
